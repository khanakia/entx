# testenttui

The runnable **demo + test bed** for the [`enttui`](../enttui) library — a
schema-driven terminal UI generator for [ent](https://entgo.io).

`enttui` itself is a pure library: no Taskfile, no example, no `main`.
Everything you need to *see it work* and *test it* lives here.

## Why this module exists

Two jobs:

1. **Demo** — `task run` boots a seeded in-memory SQLite database and
   launches the full TUI so you can browse/edit the generated views.
2. **Coverage** — the schema deliberately uses **three different ID Go
   types at once** so the codegen's per-entity ID handling is exercised
   and regression-tested:

   | Entity  | ID Go type  | Source                         |
   |---------|-------------|--------------------------------|
   | `User`    | `string`    | uuid-string mixin (`idMixin`)  |
   | `Post`    | `int`       | ent default PK                 |
   | `Comment` | `int`       | ent default PK (2nd int table) |
   | `Tag`     | `uuid.UUID` | native uuid mixin (`uuidMixin`) |

   enttui's runtime is type-erased (`Row.ID` is always `string`). The
   codegen emits a per-entity `parse<Name>ID` helper so each table gets
   the correct inbound `string → ID` conversion (`identity` / `strconv`
   / `uuid.Parse`) and `fmt.Sprint` outbound. This module proves that.

## Layout

```
testenttui/
├── schema/        ent schema + enttui annotations + id mixins
├── entc.go        stage-1 generator: ent codegen      → ./ent
├── doc.go         go:generate wiring (both stages) + docs
├── ent/           GENERATED — ent client (do not edit)
├── tui/gen/       GENERATED — enttui register_*.go glue (do not edit)
├── cmd/tui/       the demo launcher (sqlite + seed + Run)
├── crud_test.go   e2e: CRUD + RegisterAll across all 3 ID types
└── tui/gen/idtypes_test.go  white-box parse<Name>ID conversion tests
```

### How `./tui/gen` is generated

It is **not** generated by code in this module — the generator lives in
the `enttui` library and is run as a tool. Two stages:

1. `go run entc.go` → ent codegen → `./ent`
2. `go run github.com/khanakia/entx/enttui/cmd/enttui --schema ./schema --out ./tui/gen --package gen --ent-pkg github.com/khanakia/entx/testenttui/ent` → `./tui/gen`

Both are wired as `//go:generate` directives in `doc.go`, so
`go generate ./...` (or `task gen`) runs them in order.

## Commands

All via [Task](https://taskfile.dev) from this directory (`task --list`):

| Command          | What it does                                              |
|------------------|-----------------------------------------------------------|
| `task gen`       | ent codegen **then** enttui register codegen              |
| `task gen:ent`   | stage 1 only (`entc.go` → `./ent`)                        |
| `task gen:tui`   | stage 2 only (enttui CLI → `./tui/gen`)                   |
| `task build`     | `go build ./...`                                          |
| `task vet`       | `go vet ./...`                                            |
| `task test`      | run the suite (ID-type conversion + CRUD round-trip)      |
| `task test:v`    | tests, verbose                                            |
| `task run`       | launch the demo TUI (in-memory sqlite, seeded)            |
| `task run -- --kind user --view list` | run with flags                       |
| `task tidy`      | `go mod tidy`                                             |
| `task clean`     | delete `./ent` + `./tui/gen`                              |
| `task regen`     | clean → gen → tidy → build → test (from-scratch verify)   |

From the repo root: `task -d testenttui <task>`.

## Quick start

```bash
cd testenttui
task regen        # clean-room: regenerate everything, build, test
task run          # browse the seeded demo in your terminal
```

## Editing the schema

1. Edit `schema/*.go` (entities, fields, edges, `enttui.*` annotations,
   or swap an ID mixin to change a table's ID type).
2. `task gen` to regenerate `./ent` + `./tui/gen`.
3. `task test` / `task run`.

`./ent` and `./tui/gen` are generated — never hand-edit; rerun `task gen`.

## Tests

- `tui/gen/idtypes_test.go` — white-box, calls the generated
  `parseUserID` / `parsePostID` / `parseTagID` / `parseCommentID` and
  asserts the right conversion + error handling per type.
- `crud_test.go` — opens SQLite, creates one row per entity, asserts the
  ent ID Go types at compile time (`var _ string = u.ID`, `var _ int =
  p.ID`, `var _ uuid.UUID = tg.ID`), round-trips them, and runs
  `gen.RegisterAll` as a registration smoke test.

`task test` must stay green — it's the regression guard for the
per-entity ID-type codegen.
