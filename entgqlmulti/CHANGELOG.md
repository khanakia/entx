## [Unreleased]

### Added

- **End-to-end integration harness** at `testentgqlmulti/` inside the
  `entx` workspace. Generates real ent + entgql + entgqlmulti code,
  runs gqlgen on the per-API schemas, wires everything to an in-memory
  SQLite database, and executes real GraphQL-over-HTTP requests against
  three APIs (dashboard full-CRUD, read-only public, narrow mobile
  projection). 15 tests across 4 files — 12 runtime tests and 3
  structural SDL assertions. See `testentgqlmulti/TESTS.md` for the
  catalog.

### Fixed

- **`scalar [ID!]` leak in generated SDL.** `entgql` occasionally
  produces `ast.Type` values where the full SDL fragment is stuffed
  into `NamedType` (e.g. `NamedType: "[ID!]"`, `Elem: nil`) instead
  of using a proper list wrapper. `resolveTypeName` returned the raw
  string, `trackReferencedType` didn't match the built-in `"ID"`
  fast-path, and the fallback branch added `"[ID!]"` as a synthesized
  scalar. The resulting `scalar [ID!]` line was rejected by gqlgen.
  Added `stripTypeSyntax` to unwrap `[…]` and trailing `!` markers
  before lookup, so the inner type is resolved correctly and the
  built-in fast-path fires.
- **camelCase entries in `ApiTarget.Fields` silently dropped.** ent's
  `camel()` lowercases already-camelCase input (`"firstName"` →
  `"firstname"`), so targets that specified field names in camelCase
  never matched the SDL field names and were filtered out of the
  subset type. Pipe through `snake()` first, then `camel()`, so both
  `"firstName"` and `"first_name"` normalize to the same GraphQL form.
  Applies to `normalizeFieldNames` (entity + mutation-input filtering)
  and `filterWhereInputFields` (WhereInput field filtering).
- **Orphan edge predicates broke gqlgen compile in subset APIs.**
  `UserWhereInput` copied into a subset API retained fields like
  `hasPostsWith: [PostWhereInput!]` even when `Post` wasn't exported
  to that API, leaving gqlgen with an undefined type reference. Added
  `pruneOrphanWhereInputFields` that runs after all types are added:
  walks every `*WhereInput` in the output and drops any field whose
  resolved type is a `*WhereInput` not present in the schema. The
  scalar flag (`hasPosts: Boolean`) survives — only relation-input
  predicates (`hasPostsWith: [...]`) are pruned.

## [0.1.0]

### Added

- Initial release of the `entgqlmulti` ent+entgql extension. Filters
  the monolithic schema generated by `entgql.WithSchemaGenerator()`
  into per-API `.graphql` files using `entgqlmulti.ApiConfig()`
  annotations on ent schemas. Supports:
  - Multiple output APIs with distinct file paths.
  - Per-API field whitelisting via `ApiTarget.Fields` (snake_case or
    camelCase).
  - Subset types with custom `TypeName` bound to the underlying ent
    struct via `@goModel`.
  - Root-level `Query` connection fields, optionally with `Filters`
    (WhereInput) and `OrderBy` (Order input + OrderField enum).
  - Per-API `Mutations` (Create/Update input + mutation fields),
    optionally restricted to the target's `Fields` whitelist.
  - Multiple `ApiTarget`s per entity per API (two GraphQL types
    generated from the same ent struct, e.g. `Chatbot` +
    `ChatbotSummary` both backed by `ent.Chatbot`).
  - `QueryName` override for the root query field name.
  - Edge-field filtering: edges to entities not exported in this API
    are stripped from the emitted type.
  - Automatic inclusion of referenced scalars, enums, and the common
    Relay types (`Cursor`, `PageInfo`, `OrderDirection`, `Node`).
