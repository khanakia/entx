# Mutations reference

`entpoly` matches Laravel's Eloquent surface for every mutation verb on polymorphic relationships. This page is a Laravel-to-ent translation table; pick the row that matches what you want to do and copy the right column into your codebase.

## Setting the parent on a child (`MorphTo`)

| Laravel | ent + entpoly |
|---|---|
| `$comment->commentable()->associate($post)` | `client.Comment.UpdateOneID(c.ID).SetCommentable(post).Save(ctx)` |
| Same, on create | `client.Comment.Create().SetBody("hi").SetCommentable(post).Save(ctx)` |
| `$comment->commentable()->dissociate()` | `client.Comment.UpdateOneID(c.ID).ClearCommentable().Save(ctx)` |
| Replace one parent with another | `client.Comment.UpdateOneID(c.ID).SetCommentable(video).Save(ctx)` |

`SetCommentable(p Morphable)` and `ClearCommentable()` are generated by `entpoly` for every `Create` / `Update` / `UpdateOne` / `Mutation` builder of the child type. They wrap the underlying `SetCommentableID` / `SetCommentableType` setters that ent generates from the morph columns; you can still call those raw setters when you need to bypass the `Morphable` interface (for example, when you only have an id and a type string in hand from an external system).

## Adding children to a parent (`MorphMany`)

There is no `parent.AddChildren(...)` mutator on parents that own children via `MorphMany`, because ent's parent-side mutation builders are wired to true edges and the polymorphic relation is not a true edge. Two equivalent patterns, take your pick:

### Pattern A — set on the child

```go
// Equivalent to $post->comments()->save($comment).
_, _ = client.Comment.UpdateOneID(c.ID).SetCommentable(post).Save(ctx)

// Equivalent to $post->comments()->create([...]).
_, _ = client.Comment.Create().SetBody("hi").SetCommentable(post).Save(ctx)
```

### Pattern B — bulk via builder list

```go
// Equivalent to $post->comments()->saveMany([$a, $b, $c]).
_, _ = client.Comment.MapCreateBulk(rows, func(c *ent.CommentCreate, i int) {
    c.SetBody(rows[i].body).SetCommentable(post)
}).Save(ctx)
```

`MapCreateBulk` is a vanilla ent builder; entpoly does not need to wrap it.

## Many-to-many: attach / detach / sync / toggle

Polymorphic M2M is implemented via a pivot entity. Mutations operate on pivot rows directly, not on the holder's `posts`/`videos` association methods. The runtime helpers in `entpoly/helper` exist for exactly this reason — they compute the set diffs and leave the database calls to your typed pivot builders.

### Attach

Laravel:

```php
$tag->posts()->attach($post, ['added_by' => 'aman']);
```

ent + entpoly:

```go
_, _ = client.Taggable.Create().
    SetTagID(tag.ID).
    SetTaggable(post).
    SetAddedBy("aman").
    Save(ctx)
```

### Detach

Laravel:

```php
$tag->posts()->detach($post);
```

ent + entpoly:

```go
_, _ = client.Taggable.Delete().
    Where(
        taggable.TagID(tag.ID),
        taggable.TaggableIDEQ(post.MorphID()),
        taggable.TaggableTypeEQ("post"),
    ).
    Exec(ctx)
```

### Sync — replace the whole attached set

Laravel:

```php
$tag->posts()->sync($postIDs);
```

ent + entpoly:

```go
import "github.com/khanakia/entx/entpoly/helper"

// 1. Read currently-attached parent IDs.
attached, _ := client.Taggable.Query().
    Where(taggable.TagID(tag.ID), taggable.TaggableTypeEQ("post")).
    Select(taggable.FieldTaggableID).Strings(ctx)

// 2. Compute the diff.
postIDs := []string{post1.MorphID(), post2.MorphID()}
toAttach, toDetach := helper.Sync(attached, postIDs)

// 3. Apply the diff.
for _, id := range toAttach {
    _, _ = client.Taggable.Create().
        SetTagID(tag.ID).
        SetTaggableID(id).
        SetTaggableType("post").
        Save(ctx)
}
if len(toDetach) > 0 {
    _, _ = client.Taggable.Delete().
        Where(
            taggable.TagID(tag.ID),
            taggable.TaggableTypeEQ("post"),
            taggable.TaggableIDIn(toDetach...),
        ).
        Exec(ctx)
}
```

### Sync without detaching

Laravel:

```php
$tag->posts()->syncWithoutDetaching($postIDs);
```

ent + entpoly:

```go
toAttach := helper.SyncWithoutDetach(attached, postIDs)
for _, id := range toAttach {
    _, _ = client.Taggable.Create().SetTagID(tag.ID).SetTaggableID(id).SetTaggableType("post").Save(ctx)
}
```

### Toggle

Laravel:

```php
$tag->posts()->toggle($postIDs);
```

ent + entpoly:

```go
toAttach, toDetach := helper.Toggle(attached, postIDs)
// Apply with Create / Delete on the pivot as above.
```

### Update pivot extras

Laravel:

```php
$tag->posts()->updateExistingPivot($postID, ['sort_order' => 5]);
```

ent + entpoly — the pivot is a real ent type, so update it directly:

```go
_, _ = client.Taggable.Update().
    Where(
        taggable.TagID(tag.ID),
        taggable.TaggableIDEQ(post.MorphID()),
        taggable.TaggableTypeEQ("post"),
    ).
    SetSortOrder(5).
    Save(ctx)
```

## Cascade deletes

Polymorphic columns do not carry FK constraints, so deleting a parent row will *not* automatically delete its polymorphic children. Two ways forward:

1. Delete children explicitly before deleting the parent. The cleanest place is a transaction or a deletion-orchestration helper that knows which polymorphic relations point at the type being deleted.
2. Pair `entpoly` with the [`entcascade`](../../entcascade) extension to generate application-level cascade-delete helpers. (v2 of `entpoly` will optionally emit these directly when both extensions are registered.)

```go
// Manual cascade.
_, _ = client.Comment.Delete().
    Where(comment.CommentableIDEQ(post.MorphID()), comment.CommentableTypeEQ("post")).
    Exec(ctx)
_, _ = client.Post.DeleteOneID(post.ID).Exec(ctx)
```

## Touch parents on child save

Laravel automatically bumps `updated_at` on the parent when a child is saved if the child declares `protected $touches = ['commentable']`. `entpoly` ships the same behaviour via `.Touch()` on the `MorphTo` edge builder:

```go
// ent/schema/comment.go
func (Comment) Edges() []ent.Edge {
    return []ent.Edge{
        entpoly.MorphTo("commentable", Post.Type, Video.Type).
            Touch(),                  // bump parent.updated_at
        // or:
        entpoly.MorphTo("commentable", Post.Type, Video.Type).
            Touch("modified_at"),     // bump a custom field name
    }
}
```

Wire the runtime hook once at startup:

```go
client := ent.NewClient(...)
ent.RegisterPolyHooks(client)        // installs every Required / Touch hook
```

Every successful Save of a `Comment` (OpCreate / OpUpdate / OpUpdateOne) now bumps the polymorphic parent's `updated_at`. Hook ordering: child save runs first; only on success does the touch fire. A touch failure rolls the whole save back, matching Laravel.

Each parent listed in the `MorphTo` must declare the touched field — typically `field.Time("updated_at").Default(time.Now).UpdateDefault(time.Now)`. Missing field = compile error pointing at the missing `Set<Field>` method in `polymorphic.go`.

## Required relations (forbid unset)

```go
entpoly.MorphTo("commentable", Post.Type, Video.Type).Required()
```

The generated hook (also wired by `RegisterPolyHooks`) rejects:

- Create without `SetCommentable(...)` — the discriminator pair is unset
- Update / UpdateOne that calls `ClearCommentable()` — relation is required

Pairs with `.Touch()` cleanly — install both in the same `RegisterPolyHooks` call.

## Cascade delete

Polymorphic columns cannot carry FK constraints, so the database will not cascade for us. `.Cascade()` installs a pre-delete hook on every allowed parent that removes the child rows pointing at it:

```go
entpoly.MorphTo("commentable", Post.Type, Video.Type).Cascade()
```

When a `Post` is deleted, every `Comment` with `commentable_type='post'` is deleted in the same logical operation. The hook runs BEFORE the parent delete so no orphan window opens. Child errors propagate and abort the parent delete.

Combines with `.Required()` and `.Touch()` on the same edge — register all three hooks via the single `RegisterPolyHooks(client)` call:

```go
entpoly.MorphTo("commentable", Post.Type, Video.Type).
    Required().
    Touch("updated_at").
    Cascade()
```

```go
// main.go
client := ent.NewClient(...)
ent.RegisterPolyHooks(client)   // installs every hook declared above
```
