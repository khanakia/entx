# Codegen pipeline

How the `enttui` CLI turns an ent schema directory into Go code.

## Overview

```
   ./schema/                          ./tui/gen/
   ┌───────────────┐                  ┌───────────────────────┐
   │ task.go       │                  │ register_task.go      │
   │ memory.go     │   enttui gen     │ register_memory.go    │
   │ decision.go   │  ─────────────▶  │ register_decision.go  │
   │ …             │                  │ …                     │
   │               │                  │ register_all.go       │
   └───────────────┘                  └───────────────────────┘
```

One CLI invocation walks the schema, applies conventions + annotations (M1), and emits one `register_<name>.go` per browsable entity plus a `register_all.go` aggregator. Output is gofmt'd; comments mark each file `// Code generated by enttui. DO NOT EDIT.`.

## Pipeline stages

### 1. Schema load

`codegen.Generate` calls `entc.LoadGraph(schemaPath, &gen.Config{...})`. Under the hood, entc:

- Spawns a temporary Go program that imports your schema package.
- Reflects every `ent.Schema` type — fields (including mixin-inherited ones), edges, indexes, annotations.
- Returns a `*gen.Graph` whose `.Nodes []*gen.Type` is everything we care about.

We pass a stub `Target` / `Package` to the config — we don't need any of entc's own output, just the parsed graph.

### 2. Per-entity extraction → `EntityMeta`

For each `*gen.Type` in the graph we call `extractEntity(n, opts, kindByType)`:

1. **Skip checks:**
   - `n.ID == nil` → no id → not browsable.
   - `isInternal(n.Name)` → tables like `SchemaMigration`, `AuditLog`, FTS shadow tables.
   - `opts.Skip[n.Name]` → user opt-out via `--skip A,B,C`.

   Scope predicates are **independent** of inclusion. For every name in `opts.ScopeFields` (e.g. `project_id`, `tenant_id`) the codegen records the `{Key, GoName}` pair for any matching field on the entity. The template emits one predicate per match using `gen.Field.StructField()` for the Go method name — `project_id` → `ProjectID`, `tenant_id` → `TenantID`, etc. Entities without any configured scope field stay browsable, just unscoped.

2. **Field iteration** (Type.ID + Type.Fields combined):
   - Match each field name against `opts.ScopeFields` → append `{Key, GoName}` to `EntityMeta.ScopeFields`.
   - Sniff `created_at`, `updated_at` → boolean flags.
   - First match on `title` / `name` / `display_name` → `TitleField`.
   - First match on `body` / `description` / `content` → `BodyField`.
   - First enum match on `status` / `severity` / `kind` / `state` → `StatusField`.
   - Every other non-internal field → appended to `Columns`.
   - String fields named `title|name|body|description` → added to `FilterPredicates` for substring search.

3. **Field kind discrimination** via `fieldKind(*gen.Field)` → one of `string`, `stringPtr`, `enum`, `enumPtr`, `time`, `timePtr`, `scalar`, `scalarPtr`. The template switches on this string to emit the right accessor body.

4. **Edge iteration:**
   - Each edge whose target type is itself browsable → an `EdgeMeta`.
   - `e.Unique` → `EdgeUpward` (jump to parent); else `EdgeDrill` (open child list).
   - The Go method name on the client (`client.Post.QueryAuthor`) comes from `e.StructField()` — never `strings.Title(e.Name)`, which would mangle `assemble_run` → `Assemble_run`.
   - Every edge gets a single-char letter trigger picked from its name (reserved: `h j k l g n p r q v y s` — vim top-level keys). No magic "first drill claims enter" — enter always means "open preview".

5. **Enum values.** For ent `field.Enum(...)` fields the codegen pulls `f.EnumValues()` into `FieldMeta.EnumValues` and emits it into the generated `Column{EnumValues: []string{...}}` literal. Filterable enums also get a `case runtime.OpIn, runtime.OpNotIn` branch in the dispatch that splits the `|`-joined `FilterCondition.Value` and calls `pred.<Field>In(...)` / `<Field>NotIn(...)`. The `"strings"` import is gated on `EntityMeta.NeedsStrings`.

6. **Related columns** (from `enttui.RelatedColumns` annotation):
   - Each `{Edge, Field, Label}` entry → a `FieldMeta` with `Kind: "related"` carrying `EdgeGoName`, `TargetTitleField`, `TargetTitleKind`, plus filter-dispatch metadata (`HasEdgeMethod`, `TargetPredAlias`).
   - The edge's `StructField()` is appended to `EntityMeta.WithLoaders` (deduped), causing the template to emit `q = q.With<Edge>()` so the relation is eager-loaded — no N+1.
   - The target type's predicate package is registered in `EntityMeta.RelatedImports` (one aliased import per unique target type). Used by both the filter and sort dispatch.
   - **Display**: `valueExpr`'s `related` branch emits `r.Edges.<EdgeGoName>.<TargetTitleField>` with nil-checks (extra deref guard for `stringPtr` targets).
   - **Filter** (`Filterable: true`): the dispatch's `related` branch emits `pred.Has<Edge>With(targetPred.<Field><Op>(v))` for `OpEq` / `OpNeq` / `OpContains`. Sub-select semantics, no SELECT-list mangling.
   - **Sort** (`Sortable: true`): the multi-sort dispatch's `related` branch emits `pred.By<Edge>Field(targetPred.Field<GoName>, sql.OrderAsc())` using ent's generated edge-order helper. Toggles `EntityMeta.NeedsSQL` so the file imports `entgo.io/ent/dialect/sql`.
   - Unknown edge name or unknown target field → entry is silently dropped.

7. **Form support** (from `enttui.Editable{}` per-field + entity-level `AllowCreate{}` / `AllowDelete{}`):
   - Each `Editable{}` field → a `FormField` entry (Key, Label, Kind, Required, EnumValues) plus a setter line in the generated `Update` closure (`u.SetX(v)`; pointer fields get `Clear<F>()` on empty input; enum fields cast via the pre-computed `EnumGoTypeCast`).
   - `AllowCreate{}` → a `Create` closure (`client.X.Create().SetX(v)…Save`), scope-keyed setters injected from `opts`.
   - `AllowDelete{}` → a `Delete` closure (`client.X.DeleteOneID(id).Exec`).
   - `Required` mirrors `!f.Optional && !f.Nillable && name != "id"`.

8. **Bulk-copy / export flags** (from entity-level `AllowBulkCopy{}` / `AllowExport{}`): emitted as plain `AllowBulkCopy: true` / `AllowExport: true` booleans on the spec literal. No closures — the runtime owns selection + format + file-write; codegen only flips the gates.

9. **`NeedsFmt` flag.** Set true if any column accessor renders a non-string scalar — gates the `"fmt"` import. (`NeedsStrings` / `NeedsSQL` similarly gate `strings` / `entgo.io/ent/dialect/sql` for enum-In and related-sort dispatch.)

### 3. Template rendering

Two templates live in `enttui/codegen/templates/`, embedded via `//go:embed templates/*.tmpl`:

- **`entity.tmpl`** — emits one register function per entity. Uses a nested `valueExpr` template to emit the body of each field accessor, dispatching on the `Kind` string.
- **`register_all.tmpl`** — emits the aggregator. Imports only `ent` + `runtime`; calls every per-entity register in alphabetical order.

Each entity is written to `register_<lowercase-name>.go`. The aggregator is `register_all.go`.

### 4. Format pass

Every rendered file goes through `go/format.Source` before being written. If gofmt fails (template bug) the raw output is written to `<file>.unformatted` and the generator returns an error pointing at it.

## Output shape — example

For an arbitrary ent type `Post` the codegen emits something like:

```go
// Code generated by enttui. DO NOT EDIT.

package tuigen

import (
    "context"
    "encoding/json"

    "myproject/ent"
    entPost "myproject/ent/post"

    "enttui/runtime"
)

// registerPost wires *ent.Post into the enttui runtime.
func registerPost(app *runtime.App, client *ent.Client) {
    runtime.Register(app, runtime.EntitySpec[*ent.Post]{
        Kind:     "post",
        Display:  "Posts",
        Group:    "data",
        Icon:     "•",
        PageSize: 200,

        Fetch: func(ctx context.Context, opts runtime.ListOpts) ([]*ent.Post, int, error) {
            q := client.Post.Query()
            // … scope predicates, structured filters, multi-sort dispatch …
            total, err := q.Clone().Count(ctx)
            if err != nil { return nil, 0, err }
            rows, err := q.Offset(opts.Offset).Limit(opts.Limit).All(ctx)
            return rows, total, err
        },

        // Ent-native serialization — used by the `yj` clipboard shortcut.
        JSON: func(r *ent.Post) ([]byte, error) { return json.Marshal(r) },

        Columns: []runtime.Column[*ent.Post]{
            {Key: "id", Label: "Id", Get: func(r *ent.Post) string { return r.ID }},
            {Key: "title", Label: "Title", Get: func(r *ent.Post) string { return r.Title }},
            {Key: "status", Label: "Status", Get: func(r *ent.Post) string { return string(r.Status) }},
            {Key: "published_at", Label: "Published At", Get: func(r *ent.Post) string {
                if r.PublishedAt == nil || r.PublishedAt.IsZero() { return "" }
                return r.PublishedAt.Format("2006-01-02 15:04:05")
            }},
            // … more columns
        },

        Edges: []runtime.EdgeSpec[*ent.Post]{
            {
                Name: "author", Display: "→ Authors", Kind: runtime.EdgeUpward, Trigger: "a",
                ResolveUpward: func(ctx context.Context, r *ent.Post) (runtime.EntityRef, error) {
                    tgt, err := client.Post.QueryAuthor(r).Only(ctx)
                    if err != nil { return runtime.EntityRef{}, err }
                    return runtime.EntityRef{Kind: "author", ID: tgt.ID}, nil
                },
            },
            // … more edges
        },
    })
}
```

Plus `register_all.go`:

```go
// Code generated by enttui. DO NOT EDIT.

package tuigen

import (
    "myproject/ent"
    "enttui/runtime"
)

func RegisterAll(app *runtime.App, client *ent.Client) {
    registerAuthor(app, client)
    registerComment(app, client)
    registerPost(app, client)
    // … one line per browsable entity, alphabetical
}
```

## Determinism

- Entity iteration is `sort.SliceStable` by `Name` — same input → same output, every time.
- Edge iteration follows ent's own ordering.
- No timestamps, hostnames, or random IDs in generated files.

This means generated files diff cleanly when reviewed in source control. If a regen produces a noisy diff, you almost certainly have a non-deterministic bug in the codegen — file an issue.

## Customizing the generator

If you need shapes the default template doesn't emit (custom predicates, additional metadata fields), you have three options:

1. **Annotate the schema.** Adds metadata that the existing templates already consume. Lowest friction.
2. **Fork `entity.tmpl`.** Vendor `enttui/codegen/templates/` into your project and run codegen with a `--template-dir` override (not yet supported — open issue if needed).
3. **Hand-write `register_<name>.go` for one entity.** Add it to `--skip` so codegen leaves it alone, then maintain it manually. Other entities continue to regenerate normally.
